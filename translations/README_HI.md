# ⚖️ PEFL - व्यक्तिगत और शैक्षिक निःशुल्क लाइसेंस

[![License: PEFL v1.0](../badges/pefl-badge-red.svg)](../LICENSE.hi)
[![License: PEFL v1.0 (Dark)](../badges/pefl-badge-dark.svg)](../LICENSE.hi)

🇬🇧 **[Read this README in English](../README.md)**

**व्यक्तिगत और शैक्षिक निःशुल्क लाइसेंस (PEFL)** एक आधुनिक, एकल-लेखक कस्टम लाइसेंस है जिसे सॉफ्टवेयर परियोजनाओं को अनधिकृत व्यावसायिक और कॉर्पोरेट शोषण से बचाने के लिए डिज़ाइन किया गया है। यह व्यक्तिगत, शैक्षणिक और गैर-लाभकारी उपयोग के लिए 100% निःशुल्क पहुंच प्रदान करता है, जबकि व्यावसायिक संस्थाओं को लिखित प्राधिकरण प्राप्त करने और लाइसेंस शुल्क का भुगतान करने की आवश्यकता होती है।

इसके अतिरिक्त, उपयोगकर्ताओं को सॉफ़्टवेयर का उपयोग करने से पहले शर्तों के बारे में स्पष्ट रूप से जागरूक करने के लिए PEFL में एक अद्वितीय **अनिवार्य उपयोगकर्ता-अनुमोदन खंड** है।

## ⚙️ अपने प्रोजेक्ट पर PEFL कैसे लागू करें

अपने प्रोजेक्ट पर PEFL लागू करने में पांच सरल कदम लगते हैं:

### 1. लाइसेंस फ़ाइलें कॉपी करें
अपने रिपॉजिटरी के रूट डायरेक्टरी में [LICENSE](../LICENSE) फ़ाइल (और/या ऊपर सूचीबद्ध किसी भी अनुवादित पाठ) को जोड़ें। साथ ही सारांश फ़ाइल [NOTICE](../templates/NOTICE.md) भी जोड़ें।

### 2. फ़ाइल हेडर जोड़ें
सभी स्रोत फ़ाइलों की शुरुआत में आधिकारिक हेडर नोटिस जोड़ें।
- **अंग्रेजी**: [header.txt](../templates/header.txt)
- **इतालवी**: [header_IT.txt](../templates/header_IT.txt)

### 3. खंड 3 को पूरा करें (स्टार्टअप पर EULA की जाँच)
पीईएफएल का खंड 3 कहता है:
> *"यह लाइसेंस सॉफ़्टवेयर के प्रत्येक लॉन्च पर हमेशा अग्रभूमि में दिखाई देना चाहिए (या कम से कम हर 7 दिनों में एक बार यदि स्वीकृति तिथि को सुरक्षित रूप से संग्रहीत करने के लिए एक स्थानीय दृढ़ता तंत्र उपलब्ध है), और किसी भी उपयोग से पहले उपयोगकर्ता द्वारा स्पष्ट रूप से अनुमोदित होना चाहिए।"*

इस शर्त को पूरा करने के लिए, आपके सॉफ़्टवेयर को शुरू करने से पहले उपयोगकर्ता को लाइसेंस स्वीकार करने के लिए संकेत देना चाहिए। [templates/](../templates/) फ़ोल्डर में हमारे आधिकारिक टेम्प्लेट में से एक का उपयोग करें:
- **Bash / Linux / Unix**: [pefl-license-approval-cli.sh](../templates/pefl-license-approval-cli.sh)
- **Zsh / macOS**: [pefl-license-approval-cli-macos.zsh](../templates/pefl-license-approval-cli-macos.zsh)
- **PowerShell (Windows/क्रॉस-प्लेटफ़ॉर्म)**: [pefl-license-approval-cli.ps1](../templates/pefl-license-approval-cli.ps1)
- **Windows CMD Batch**: [pefl-license-approval-cli.bat](../templates/pefl-license-approval-cli.bat)
- **Python / कंसोल**: [pefl-license-approval-cli.py](../templates/pefl-license-approval-cli.py)
- **JavaScript / वेब**: [pefl-license-approval-web.js](../templates/pefl-license-approval-web.js)

### 4. पैकेज मैनेजर कॉन्फ़िगरेशन
[templates/package-metadata.md](../templates/package-metadata.md) निर्देशों का उपयोग करके npm (`package.json`), Python (`pyproject.toml`) या Cargo (`Cargo.toml`) में अपने पैकेज की मालिकाना स्थिति घोषित करें।

### 5. बैज दिखाएं
विजिटर को पीईएफएल शर्तों को स्पष्ट रूप से घोषित करने के लिए अपने रिपॉजिटरी की README.md फ़ाइल में आधिकारिक बैज जोड़ें।

## ❓ अक्सर पूछे जाने वाले प्रश्न (FAQ)

### क्या PEFL एक मानक ओपन सोर्स लाइसेंस है?
नहीं। PEFL एक OSI-अनुमोदित लाइसेंस नहीं है। चूंकि यह व्यावसायिक उपयोग को प्रतिबंधित करता है, इसे "Source-Available" या "Fair-Code" लाइसेंस के रूप में वर्गीकृत किया गया है।

### क्या मैं किसी PEFL प्रोजेक्ट को फोर्क कर सकता हूँ?
हाँ, आप अपने व्यक्तिगत उपयोग के लिए PEFL-लाइसेंस प्राप्त सॉफ़्टवेयर को फोर्क और संशोधित करने के लिए स्वतंत्र हैं। हालांकि, खंड 5 लेखक की लिखित सहमति के बिना संशोधित कोड को एक अलग लाइसेंस के तहत पुनर्वितरित करने से रोकता है।
