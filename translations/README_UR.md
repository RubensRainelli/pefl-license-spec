# ⚖️ PEFL - شخصی اور تعلیمی مفت لائسنس

[![License: PEFL v1.0](../badges/pefl-badge-red.svg)](../LICENSE.ur)
[![License: PEFL v1.0 (Dark)](../badges/pefl-badge-dark.svg)](../LICENSE.ur)

🇬🇧 **[Read this README in English](../README.md)**

**شخصی اور تعلیمی مفت لائسنس (PEFL)** ایک جدید، واحد مصنف کا اپنی مرضی کے مطابق لائسنس ہے جسے سافٹ ویئر پروجیکٹس کو غیر مجاز تجارتی اور کارپوریٹ استحصال سے بچانے کے لیے ڈیزائن کیا گیا ہے۔ یہ شخصی، تعلیمی اور غیر منافع بخش استعمال کے لیے 100% مفت رسائی فراہم کرتا ہے، جب کہ تجارتی اداروں کو تحریری اجازت حاصل کرنے اور لائسنس فیس ادا کرنے کی ضرورت ہوتی ہے۔

مزید برآں، سافٹ ویئر کو استعمال کرنے سے پہلے صارفین کو شرائط کے بارے میں واضح طور پر آگاہ کرنے کے لیے PEFL میں ایک منفرد **لازمی صارف کی منظوری کا طریقہ کار** ہے۔

## ⚙️ اپنے پروجیکٹ پر PEFL کو کیسے لاگو کریں۔

اپنے پروجیکٹ پر PEFL لاگو کرنے میں پانچ آسان اقدامات لگتے ہیں:

### 1. لائسنس کی فائلیں کاپی کریں
اپنے ذخیرے کی روٹ ڈائرکٹری میں [LICENSE](../LICENSE) فائل (اور/یا اوپر درج کوئی بھی ترجمہ شدہ متن) شامل کریں۔ تفصیلی خلاصہ فائل [NOTICE](../templates/NOTICE.md) بھی شامل کریں۔

### 2. فائل ہیڈرز شامل کریں
تمام سورس فائلوں کے آغاز میں آفیشل ہیڈر نوٹس شامل کریں۔
- **انگریزی**: [header.txt](../templates/header.txt)
- **اطالوی**: [header_IT.txt](../templates/header_IT.txt)

### 3. کلاز 3 کو پورا کریں (اسٹارٹ اپ پر EULA کی جانچ)
پی ای ایف ایل کا کلاز 3 کہتا ہے:
> *"یہ لائسنس سافٹ ویئر کے ہر آغاز پر ہمیشہ پیش منظر میں نظر آنا چاہیے (یا کم از کم ہر 7 دن میں ایک بار اگر قبولیت کی تاریخ کو محفوظ طریقے سے اسٹور کرنے کے لیے مقامی تسلسل کا طریقہ کار دستیاب ہو)، اور کسی भी استعمال سے پہلے صارف کی طرف سے واضح طور پر منظور شدہ ہونا چاہیے۔"*

اس شرط کو پورا کرنے کے لیے، آپ کے سافٹ ویئر کو شروع کرنے سے پہلے صارف کو لائسنس قبول کرنے کے لیے اشارہ کرنا چاہیے۔ [templates/](../templates/) فولڈر میں ہمارے آفیشل ٹیمپلیٹس میں سے ایک کا استعمال کریں:
- **Bash / Linux / Unix**: [pefl-license-approval-cli.sh](../templates/pefl-license-approval-cli.sh)
- **Zsh / macOS**: [pefl-license-approval-cli-macos.zsh](../templates/pefl-license-approval-cli-macos.zsh)
- **PowerShell (ونڈوز/کراس پلیٹ فارم)**: [pefl-license-approval-cli.ps1](../templates/pefl-license-approval-cli.ps1)
- **Windows CMD Batch**: [pefl-license-approval-cli.bat](../templates/pefl-license-approval-cli.bat)
- **Python / کنسول**: [pefl-license-approval-cli.py](../templates/pefl-license-approval-cli.py)
- **JavaScript / ویب**: [pefl-license-approval-web.js](../templates/pefl-license-approval-web.js)

### 4. پیکیج مینیجر کی ترتیب
[templates/package-metadata.md](../templates/package-metadata.md) ہدایات کا استعمال کرتے ہوئے npm (`package.json`)، Python (`pyproject.toml`) یا Cargo (`Cargo.toml`) میں اپنے پیکیج کی ملکیتی حیثیت کا اعلان کریں۔

### 5. شیلڈز ڈسپلے کریں
شائقین کو پی ای ایف ایل شرائط واضح طور پر بتانے کے لیے اپنے ذخیرے کی README.md فائل میں آفیشل بیج شامل کریں۔

## ❓ اکثر پوچھے جانے والے سوالات (FAQ)

### کیا PEFL ایک معیاری اوپن سورس لائسنس ہے؟
نہیں۔ PEFL ایک OSI سے منظور شدہ لائسنس نہیں ہے۔ چونکہ یہ تجارتی استعمال کو محدود کرتا ہے، اس لیے اسے "Source-Available" یا "Fair-Code" لائسنس کے طور پر درجہ بندی کیا گیا ہے۔

### کیا میں کسی PEFL پروجیکٹ کو فورک کر سکتا ہوں؟
جی ہاں، آپ اپنے شخصی استعمال کے لیے PEFL لائسنس یافتہ سافٹ ویئر کو فورک اور ترمیم کرنے کے لیے آزاد ہیں۔ تاہم کلاز 5 مصنف کی تحریری رضامندی کے بغیر ترمیم شدہ کوڈ کو کسی دوسرے لائسنس کے تحت دوبارہ تقسیم کرنے سے روکتا ہے۔
