# ⚖️ PEFL - Персональная и образовательная бесплатная лицензия

[![License: PEFL v1.0](../badges/pefl-badge-red.svg)](../LICENSE.ru)
[![License: PEFL v1.0 (Dark)](../badges/pefl-badge-dark.svg)](../LICENSE.ru)

🇬🇧 **[Read this README in English](../README.md)**

**Персональная и образовательная бесплатная лицензия (PEFL)** — это современная индивидуальная лицензия от одного автора, предназначенная для защиты программных проектов от несанкционированного коммерческого и корпоративного использования. Она предоставляет 100% бесплатный доступ для личного, академического и некоммерческого использования, требуя при этом от коммерческих организаций получения письменного разрешения и уплаты лицензионного сбора.

Кроме того, PEFL содержит уникальное **обязательное условие одобрения пользователем** перед запуском программного обеспечения, чтобы гарантировать, что пользователи четко осведомлены об условиях лицензии.

## 🌟 Основная философия

Традиционные открытые лицензии позволяют многомиллиардным корпорациям коммерциализировать проекты независимых разработчиков, ничего не отдавая взамен. PEFL решает эту проблему, устанавливая четкие границы:

| Тип организации | Разрешено бесплатное использование | Требуется письменное согласие и плата |
| :--- | :---: | :---: |
| **Любители и частные лица** | 🟢 Да | ❌ Нет |
| **Студенты и исследователи** | 🟢 Да | ❌ Нет |
| **Школы и университеты** | 🟢 Да | ❌ Нет |
| **Некоммерческие организации (НКО)** | 🟢 Да | ❌ Нет |
| **Компании и бизнес-пользователи** | ❌ Нет | 🟢 Да |
| **Религиозные организации** | ❌ Нет | 🟢 Да |

## ⚙️ Как применить PEFL к вашему проекту

Применение PEFL к вашему проекту состоит из пяти простых шагов:

### 1. Скопируйте файлы лицензии
Добавьте файл [LICENSE](../LICENSE) (и/или любые локализованные переводы, перечисленные выше) в корневой каталог вашего репозитория. Также добавьте файл сводки [NOTICE](../templates/NOTICE.md).

### 2. Добавьте заголовки файлов
Добавьте официальное уведомление о заголовке в начало всех исходных файлов.
- **Английский**: [header.txt](../templates/header.txt)
- **Итальянский**: [header_IT.txt](../templates/header_IT.txt)

### 3. Выполните Условие 3 (Проверка EULA при запуске)
Условие 3 лицензии PEFL гласит:
> *«Эта лицензия должна быть всегда видна, на переднем плане, при каждом запуске Программного обеспечения (или по крайней мере один раз в 7 дней, если доступен механизм локального сохранения для безопасного хранения даты принятия), и должна быть явно одобрена пользователем перед любым использованием.»*

Чтобы удовлетворить это условие, ваше программное обеспечение должно запрашивать согласие пользователя с лицензией перед запуском. Используйте один из наших официальных шаблонов в папке [templates/](../templates/):
- **Bash / Linux / Unix**: [pefl-license-approval-cli.sh](../templates/pefl-license-approval-cli.sh)
- **Zsh / macOS**: [pefl-license-approval-cli-macos.zsh](../templates/pefl-license-approval-cli-macos.zsh)
- **PowerShell (Windows/Кроссплатформенный)**: [pefl-license-approval-cli.ps1](../templates/pefl-license-approval-cli.ps1)
- **Windows CMD Batch**: [pefl-license-approval-cli.bat](../templates/pefl-license-approval-cli.bat)
- **Python / Консоль**: [pefl-license-approval-cli.py](../templates/pefl-license-approval-cli.py)
- **JavaScript / Web**: [pefl-license-approval-web.js](../templates/pefl-license-approval-web.js)

### 4. Конфигурация пакетного менеджера
Объявите проприетарный статус вашего пакета в npm (`package.json`), Python (`pyproject.toml`) или Cargo (`Cargo.toml`), следуя нашим инструкциям в [templates/package-metadata.md](../templates/package-metadata.md).

### 5. Разместите badge
Добавьте официальный значок (badge) в README.md вашего репозитория, чтобы четко заявить об условиях PEFL посетителям.

## ❓ Часто задаваемые вопросы (FAQ)

### Является ли PEFL стандартной лицензией с открытым исходным кодом?
Нет. PEFL не является лицензией, одобренной OSI. Поскольку она ограничивает коммерческое использование, она классифицируется как лицензия класса «Source-Available» или «Fair-Code».

### Могу ли я сделать форк PEFL-проекта?
Да, вы можете свободно делать форки и изменять программное обеспечение под лицензией PEFL для своего Личного использования. Однако условие 5 запрещает распространение измененного кода под другой лицензией без письменного согласия автора.
